A flexible content card with title, description, optional eyebrow label, optional media that bleeds to the bottom edge, and an optional action area. Colors and borders use semantic tokens, so the card flips automatically in dark mode.

## Slots

| Slot      | Purpose                                                                  |
|-----------|--------------------------------------------------------------------------|
| `eyebrow` | Small label above the title. Only rendered when content is provided.    |
| `media`   | Image or video that bleeds flush to the bottom edge of the card.        |
| `button`  | Action area below description. Only rendered when content is provided.  |

All three slots use `Astro.slots.has(...)` at build time, so nothing renders (and no empty wrapper is emitted) when a slot is unused.

## Mechanism

- The eyebrow slot is wrapped in a `text-body-sm text-fg-muted` container by default. Pass richer content (a badge, icon + text) and it takes over.
- When the `media` slot has content, bottom padding drops to `pb-0` so the media sits flush. Otherwise `pb-8` keeps spacing consistent.
- The `border` prop maps to `border`, `border-y`, `border-x`, or no border — always using the theme's `--color-stroke` token.
- Title is fully optional — pass nothing and no heading element renders.

## Usage

```astro
---
import CardFeatured from "@components/CardFeatured.astro";
import Button from "@components/Button.astro";
---

<CardFeatured
  title="Design content structures your way."
  description="Full control with a streamlined, API-first experience."
  border="all"
>
  <span slot="eyebrow">Schema Builder</span>
  <img slot="media" src="/screenshot.png" alt="Schema builder interface" />
</CardFeatured>

<CardFeatured title="Build faster with our REST API." description="..." border="y" titleTag="h2">
  <span slot="eyebrow">API</span>
  <Button slot="button" href="/docs">Read docs</Button>
</CardFeatured>
```

## Notes

- The card itself is not a link — wrap it with `<Clickable>` if the whole card should navigate.
- Colors use semantic tokens, so dropping the card inside `[data-theme="dark"]` automatically flips its surface, border, and text.
