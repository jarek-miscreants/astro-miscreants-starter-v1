The "stretched link" pattern. Wraps any block of content with an absolutely-positioned `<a>` that covers the whole container, so the user can click anywhere on the card to navigate — without putting an `<a>` around the entire markup (which would nest interactive elements and break a11y).

## Mechanism

- The outer `<div>` is `position: relative` so the stretched link is contained.
- An `<a>` with `absolute inset-0 z-[1]` covers the full area of the parent.
- A `<span class="sr-only">` inside the link provides the accessible name.
- `focus-visible` styles produce a ringed outline with `rounded-[inherit]` so the focus ring matches the card's corners.
- Child content sits *below* the link in DOM order but appears unchanged visually.

## Usage

```astro
---
import Clickable from "@components/Clickable.astro";
---

<Clickable href="/blog/my-post" label="Read: My Post Title">
  <img src="/thumbnail.jpg" alt="" />
  <h3>My Post Title</h3>
  <p>A short description...</p>
</Clickable>
```

## Gotchas

- **Inner interactive elements need `position: relative; z-index: 2`** (use the `card-action` utility from `global.css`) so they sit above the stretched link and stay independently clickable.
- The `label` prop is required for accessibility — without it, the link has no accessible name.
- Don't nest two `Clickable`s — the inner stretched link will eat all clicks meant for the outer.
