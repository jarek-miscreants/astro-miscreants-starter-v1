Top-layer dialog built on native `<dialog>` + `.showModal()`. Triggers wired by id (`data-modal-open="..."`), so multiple buttons can open the same modal — common case for hero CTA + footer CTA + pricing-tier button all opening the same signup flow.

## Slots

- **`title`** — heading. Auto-wired to `aria-labelledby`.
- **`description`** — optional sub-heading paragraph. Auto-wired to `aria-describedby`.
- **default** — body content.
- **`footer`** — optional action row. Use `<Fragment slot="footer">` to drop in multiple buttons.

## Trigger / close attributes

| Attribute | Where | Behavior |
|---|---|---|
| `data-modal-open="<id>"` | Any button or link, anywhere | Opens the modal with that id |
| `data-modal-close` | Element inside the modal | Closes the enclosing dialog |
| `data-modal-close="<id>"` | Element outside, on a different modal | Closes that specific dialog |
| `data-modal-autofocus` | Child of the modal | Focused on open (overrides default first-focusable) |

## A11y

- Native `<dialog>` — keyboard, focus trap, Esc-to-close, inert background all from the platform.
- `aria-labelledby` / `aria-describedby` auto-wired to the title / description slots.
- Focus moves into the modal on open; the element with `data-modal-autofocus` wins, otherwise first focusable.
- Focus restored to the original trigger button on close — works for any close path (X, Esc, backdrop, programmatic `.close()`).
- Body scroll locked while open via `html.modal-open { overflow: hidden }`. `scrollbar-gutter: stable` on `<html>` prevents layout shift.
- `prefers-reduced-motion: reduce` disables the entry/exit animation.
