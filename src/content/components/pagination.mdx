Numeric pagination component for paginated lists, blog archives, search results, and any "page N of M" surface. Two render modes:

- **Anchor mode (default)** — pages render as real `<a href="...">` tags via the `hrefFor` prop. Works without JavaScript, plays nicely with crawlers, and supports browser middle-click / open-in-new-tab.
- **Button mode** — pages render as `<button>` and clicks dispatch a `pagination:change` event. Use this for client-side filtering or in-page list navigation.

## Anchor mode

```astro
---
import Pagination from "@components/Pagination.astro";

const currentPage = Number(Astro.url.searchParams.get("page") ?? 1);
const totalPages = Math.ceil(totalItems / pageSize);
---

<Pagination
  current={currentPage}
  total={totalPages}
  hrefFor={(p) => `?${new URLSearchParams({ page: String(p) })}`}
/>
```

Each page button is a real anchor — no JS required, works offline, plays nicely with crawlers, and supports middle-click open-in-new-tab. The current page is `aria-current="page"` and styled as the active state.

## Button mode

```astro
<Pagination current={1} total={5} buttons />

<script>
  document.addEventListener("pagination:change", (e) => {
    console.log(`Page change: ${e.detail.from} → ${e.detail.to}`);
    // Render the next page of results
  });
</script>
```

Listen on the wrapper (or any ancestor — the event bubbles). The event is `cancelable` — call `preventDefault()` to manage state externally before letting the component update its own `data-pagination-current` attribute.

## Truncation algorithm

The list always includes:
- The current page.
- `siblings` pages on each side of current.
- The first and last page (when `boundaries` is true).

Gaps between non-adjacent runs are rendered as a `…` ellipsis (`<li data-pagination-ellipsis>` with `aria-hidden`). Examples for `siblings=1`, `total=10`:

| Current | Visible pages          |
|--------:|------------------------|
| 1       | `1 2 … 10`             |
| 4       | `1 … 3 4 5 … 10`       |
| 7       | `1 … 6 7 8 … 10`       |
| 10      | `1 … 9 10`             |

## Accessibility

- `<nav aria-label="Pagination">` wrapper — screen readers announce the region.
- Current page marked with `aria-current="page"`.
- Disabled prev/next: in anchor mode, `aria-disabled="true"` (link still renders without href); in button mode, the button has the native `disabled` attribute.
- Each numeric link/button has `aria-label="Page N"` so screen readers announce purpose, not just the number.
- The `<nav>` is ordered (`<ol>`) so screen readers announce position correctly.
- Focus-visible outline uses `--color-focus`.

## Notes

- On screens narrower than 480px, the visible "Previous" / "Next" labels collapse to just the arrow icons. The `aria-label` still announces the full text.
- Use anchor mode whenever the URL is the source of truth (deep linking, sharing, SEO). Use button mode only when pagination state is in-page transient.
- For "Load more" patterns, prefer a real `<button>` with progressive disclosure — this component is for discrete page navigation, not infinite scroll.
