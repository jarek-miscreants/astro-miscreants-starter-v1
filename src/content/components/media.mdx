A small video tile that handles the three activation patterns marketing pages actually need — autoplay heroes, hover-to-play grids, and click-to-play demos — without re-implementing the lazy-load + pause-on-blur + touch-fallback logic each time.

State is exposed via `data-media-status` (`not-active` | `loading` | `playing` | `paused`) so CSS drives the poster/play-overlay swap with no extra JS.

## Status hooks

The wrapper element carries a `data-media-status` attribute you can target from CSS or JS:

| Status        | When                                                                |
|---------------|---------------------------------------------------------------------|
| `not-active`  | Mounted, not yet triggered (autoplay before in-view, click before tap). |
| `loading`     | Source assigned, waiting for `canplay`.                              |
| `playing`     | Video is playing.                                                    |
| `paused`      | Paused by hover-out, click, scroll-out, or tab-hidden.               |

The built-in styles already use these to fade the poster out and the video in, and to hide the play overlay once the video is playing.

## Slots

| Slot         | Purpose                                                                                              |
|--------------|------------------------------------------------------------------------------------------------------|
| `play-icon`  | Replace the default ▶ SVG inside the click-mode overlay.                                              |
| `toggle`     | Provide your own click target. Any `[data-media-toggle]` element inside this slot becomes the trigger. |

When a `toggle` slot is supplied, the wrapper is **not** the click target — it stops being a button and the slotted control owns the interaction.

## Mechanism

### Lazy loading

The `src` attribute on `<video>` is left empty at render time; the real URL lives on `data-media-src`. On first activation (in-view + mode trigger met), the script assigns `video.src` and calls `video.load()`. This avoids fetching every video on a grid page until they actually need to play.

### Activation modes

- **autoplay** — plays the moment the tile crosses the IntersectionObserver root margin.
- **hover** — plays on `mouseenter` / `focusin`, pauses (with `pauseDelay`) on `mouseleave` / `focusout`.
- **click** — plays on `click`. If no `toggle` slot is provided, the wrapper itself is the button (`role="button"`, `tabindex="0"`, Space/Enter handler, `aria-pressed` reflects state).

### Touch fallback

On `(hover: hover) and (pointer: fine)` = false, the resolved mode follows `touchMode` if set, otherwise:
- `hover` → `autoplay`
- `autoplay` and `click` stay as-is

### Auto-pause

- **Out of viewport** — paused (immediately) when `pauseOffscreen` is true.
- **Tab hidden** — paused on `visibilitychange` (saves CPU/battery).
- **Reduced motion** — autoplay and hover modes are gated entirely; click mode stays interactive (the user is asking for it).

### Re-init on view transitions

`initMedia()` runs once on `DOMContentLoaded` and again on every `astro:page-load`. Each instance guards against double-init via `data-script-initialized`.

## Accessibility

- Decorative video by default — `aria-hidden="true"` is set unless `alt` is provided.
- Click-mode wrapper acts as a real button: keyboard activation (Space/Enter), visible `:focus-visible` outline using `--color-focus`, and `aria-pressed` reflects play state.
- Slotted `toggle` lets you own the affordance entirely (recommended when you need a labeled button rather than a whole-tile click target).
- Autoplay and hover modes are suppressed under `prefers-reduced-motion: reduce`. Click mode is preserved because activation is user-initiated.

## Notes

- **No web/mp4 fallback array.** If you need multi-format sources, replace the inner `<video>` with `<source>` children — the lazy-load logic uses `video.src` so you'd need to adapt that, but the rest of the script works unchanged.
- **No native `controls`.** Intentional — the component is for ambient/marketing video, not a player. Wrap a real `<video controls>` if that's what you need.
- **Poster is a plain URL string**, not an `astro:assets` `ImageMetadata`. Run your asset through `astro:assets` and pass `.src` if you want optimization.
- **`muted` is hardcoded.** Browsers block sound autoplay anyway; if you genuinely need sound, switch to click mode and remove the attribute on the rendered `<video>`.
