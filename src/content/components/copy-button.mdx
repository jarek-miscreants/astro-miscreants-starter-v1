Small, focused copy-to-clipboard button. Useful next to inline tokens, share URLs, install commands — and used as the copy affordance inside `CodeBlock`.

## Slots

| Slot          | Purpose                                                              |
|---------------|----------------------------------------------------------------------|
| (default)     | Button text. Defaults to the `label` prop.                            |
| `icon-idle`   | Custom idle-state icon. Defaults to a clipboard SVG.                 |
| `icon-copied` | Custom copied-state icon. Defaults to a checkmark SVG.               |

## Accessibility

- Real `<button type="button">` — focusable, keyboard-activatable.
- `aria-label` set to `label` prop — screen readers announce the action even if the visible text is just an icon.
- A visually-hidden `aria-live="polite"` region announces "Copied to clipboard" / "Copy failed" to assistive tech.
- The state swap (idle ↔ copied) uses a `data-copy-state` attribute that CSS reads — no inline style manipulation, no flash.

## Notes

- Falls back gracefully — if the Clipboard API is denied (insecure context, permissions), the live region announces "Copy failed" and the visual state stays idle.
- `value` takes precedence over `target` if both are set.
- Reads `target.textContent.trim()` — appropriate for code/URL snippets but be aware of trailing whitespace stripping if you depend on it.
