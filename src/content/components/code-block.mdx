A pragmatic code block primitive for docs and marketing pages. Built on top of Astro's built-in `<Code />` component (so syntax highlighting is server-rendered shiki — zero client JS for highlighting), with a small client script for the copy button.

## Mechanism

- **Highlighting** — Astro's `<Code />` runs shiki at build/SSR time. No client-side highlighter is shipped to the browser.
- **Line highlight** — implemented as a one-line shiki transformer (`name: "codeblock-highlight"`) that adds `data-line-highlight="true"` to matching `<span class="line">` rows. CSS draws the highlight band.
- **Line numbers** — pure CSS counter on `.line` elements. No per-line markup needed.
- **Copy button** — the `CopyButton` component reads the literal `code` string from the prop, so what gets copied is exactly what was rendered (no whitespace surprises from copying DOM text).

## Notes

- The `<Code />` component must be imported from `astro:components` — that's the project rule (see `feedback_astro_shiki_cloudflare.md`). Don't shell out to `shiki.codeToHtml` directly; static HTML truncates on Cloudflare builds.
- Shiki appends a trailing empty line to its output. The CSS rule `.line:last-child:empty::before { content: ""; }` suppresses the number for that ghost line.
- For inline code, use the standard markdown `` `like this` `` — this component is for block-level snippets only.
