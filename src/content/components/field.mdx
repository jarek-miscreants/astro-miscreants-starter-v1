A single-input row that handles the boring-but-easy-to-get-wrong parts of form fields: associating a `<label>` to its input, wiring `aria-describedby` to helper and error text, surfacing `aria-invalid` for invalid state, and reserving space so the form doesn't jump when an error appears.

Designed to be paired with `Form`, which surfaces native validation messages into each Field's error region automatically. Works standalone too.

## Slots

For `type="select"`, the default slot can be used to write your own `<option>` elements (overrides `options`):

```astro
<Field label="Country" name="country" type="select">
  <option value="">Select country</option>
  <option value="us">United States</option>
  <option value="uk">United Kingdom</option>
</Field>
```

## Accessibility

- `<label for="…">` matched to input id.
- `aria-describedby` points at helper text, error text, or both.
- `aria-invalid="true"` set whenever an error is present (either from the `error` prop or from live validation).
- Required fields show a red `*` next to the label (decorative — `aria-hidden`); the actual signal to screen readers is the input's `required` attribute.
- The error region uses `aria-live="polite"` so changes are announced without interrupting.
- The error region reserves `min-height: 1em` so the form doesn't jump when an error message appears.
- For `checkbox` / `radio`, label and input are wrapped together in a single `<label>` so clicking either toggles the input.

## Working with Form

When wrapped in `Form`, validation messages flow automatically:

```astro
<Form action="/api/contact" method="POST">
  <Field label="Email" name="email" type="email" required />
  <button type="submit">Send</button>
</Form>
```

If the user submits with an invalid email, the Field's error region populates with the browser's native `validationMessage`, focus jumps to the first invalid input, and `aria-invalid` is set. As the user fixes the input, the error clears.

Outside a Form, the `error` prop is the only error path — set it from your own validation logic.

## Notes

- For radio groups, render multiple Field components with the same `name` and different `value` props.
- For file uploads, currently bypass Field and use a plain `<input type="file">` — file inputs have unique label/styling needs that didn't fit cleanly. May get its own component later.
- The `sr-only` class is provided by Tailwind v4 — `labelHidden` relies on it.
