A thin wrapper that fixes one job: every icon on the site shares the same size scale, the same alignment behaviour, and the same accessibility defaults. The actual graphic is whatever you drop into the slot — an `astro-icon` `<Icon>`, a raw `<svg>`, or an `<img>`. The child stretches to fill the box (`width: 100%; height: 100%`), so call sites stop carrying `h-4 w-4` / `size-5` Tailwind utilities.

## Mechanism

- Renders a single `<span class="icon icon--{size}">` that sets `width` and `height` from `--icon-size`.
- The slotted child is targeted with `:global(svg)` / `:global(img)` and stretched to `100%/100%`, so it always fills the box.
- `display: inline-flex` + `align-items: center` keeps icons baseline-aligned next to text without per-call-site flex utilities.
- `flex-shrink: 0` so an icon next to a long label can't get squashed inside a flex row.
- Accessibility: passing `label` flips on `role="img" aria-label="…"`. With no label the wrapper is `aria-hidden="true"` — the default for decorative icons.

### Fluid scaling

When `fluid` is set, `--icon-size` becomes a `clamp(MIN, intercept + slope*cqi, MAX)` expression mirroring the typography ramp in `global.css`. The icon grows when its container is wide (`cqi` = 1% of the nearest container's inline size) and pins at the min/max at the extremes. Use it only inside an element that has `container-type: inline-size` set — otherwise `cqi` resolves against the viewport and you'll just get viewport-fluid sizing.

## Usage

```astro
---
import Icon from "@components/Icon.astro";
import { Icon as LucideIcon } from "astro-icon/components";
---

{/* Decorative — aria-hidden by default */}
<Icon size="md"><LucideIcon name="lucide:search" /></Icon>

{/* Meaningful — gets role="img" + aria-label */}
<Icon size="lg" label="Open menu">
  <LucideIcon name="lucide:menu" />
</Icon>

{/* Inline SVG, no library needed */}
<Icon size="sm">
  <svg viewBox="0 0 24 24" fill="currentColor"><path d="…" /></svg>
</Icon>

{/* Fluid — scales with the surrounding container's inline size */}
<Icon size="lg" fluid><LucideIcon name="lucide:zap" /></Icon>
```

> The `astro-icon` import is aliased to `LucideIcon` because both components are named `Icon`. Only one needs an alias — pick whichever is less common in the file.

## Tokens

- Static sizes: `xs 0.75rem · sm 0.875rem · md 1rem · lg 1.25rem · xl 1.5rem`
- Fluid sizes (when `fluid` is set): `clamp()` ramps mirroring the typography scale
- Color: inherited (`currentColor`) — set via the parent's text color, e.g. `text-fg-muted` or `text-intent`
