## Slots

| Slot      | Purpose                                                              |
|-----------|----------------------------------------------------------------------|
| (default) | Tag label.                                                           |
| `icon`    | Optional leading icon (SVG). Sized to `1em` and inherits color.       |

## Events

When `closable`, clicking the close button dispatches a `tag:close` event:

```js
document.addEventListener("tag:close", (e) => {
  console.log("Removed:", e.detail.value);
});
```

The event is `cancelable` — call `preventDefault()` to keep the tag in the DOM (e.g. if you want to confirm before removing, or sync with backend state first).

## Accessibility

- Real `<button>` for close — keyboard-accessible, focus-visible outline styled to match the tag's text color.
- `aria-label` on the close button — pass a descriptive `closeLabel` so screen readers announce *what* is being removed (e.g. `"Remove News filter"`), not just "Remove".
- Color contrast: solid variants use `color-mix()` to keep text legible on tinted backgrounds; outlined variants use the foreground color directly on transparent.

## Notes

- Tag is a `<span>` (inline-flex) — drop it inside any text flow without disrupting layout.
- `warning` variant uses a literal `#d97706` (amber-600) since the project tokens don't include a warning color. Add `--color-warning` to `global.css` if you want to centralize it.
- Closable tags self-remove from the DOM after dispatch — for controlled behaviour, listen to `tag:close` and call `preventDefault()` then handle removal in your own state.
