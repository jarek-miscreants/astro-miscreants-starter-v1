A radio group of icon-only "tag" pills where the selected one expands with a bouncy spring to show its label, and the rest collapse back to icon-only. They use — native `<input type="radio">` for state, `:has(input:checked)` for styling, and the `grid-template-columns: 0fr → 1fr` trick for the width animation.

## Why no JS

| Concern              | React + Framer Motion       | Native + CSS                                                |
|----------------------|-----------------------------|-------------------------------------------------------------|
| Selection state      | `useState`                  | `<input type="radio">`                                      |
| Keyboard nav         | Manual key handlers         | Free — arrow keys move between radios                       |
| Form integration     | Custom serialization        | Submits as a normal radio under the configured `name`        |
| Layout animation     | Framer Motion `layout`      | `grid-template-columns: 0fr → 1fr` + `linear()` spring       |

## Mechanism

### `:has()` selects the active label

Every item is a `<label>` wrapping a visually-hidden `<input type="radio">` and the visible pill. Sharing a `name` makes them an exclusive group, so native arrow-key navigation works. The label's `:has(input:checked)` flips the pill's color, background, expanded label width, and `margin-inline-start` simultaneously.

### Width animation via `grid-template-columns`

The label-text wrapper is a 1-column grid. Animating `grid-template-columns` between `minmax(0, 0fr)` and `minmax(0, 1fr)` interpolates between "no track" and "auto-sized track" — animating to intrinsic width without measuring.

`minmax(0, …)` matters: without it, `white-space: nowrap` text forces the track open even at 0fr.

### The bouncy spring

A `linear()` easing function with values that overshoot 1.0 — matching `visualDuration: 0.3, bounce: 0.3` from Framer Motion. `cubic-bezier()` can't express overshoots above 1.0.

### No flex `gap`

The icon-to-label spacing is `margin-inline-start` on the label, animated in lockstep with the width — not a flex `gap`. Flex `gap` reserves space even when one item has 0 width, leaking phantom padding next to the icon on inactive pills.

## Gotchas

1. **`:has()` browser support.** Chrome 105+, Safari 15.4+, Firefox 121+. Older browsers won't apply the active-state styling — every pill looks inactive. If you need wider support, a JS shim that toggles a class on `change` would do the same job.
2. **`name` uniqueness.** Two instances with the same `name` would behave as one radio group across both. The default random `name` avoids this.
3. **Long labels.** The pill expands to label width + padding. Very long labels can wrap the row or push siblings off-screen — cap label length or constrain the parent.
